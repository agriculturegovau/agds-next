Cards are often used to curate groups of related information – for example, Top tasks. They are frequently used on landing pages to help users find what they need.

Cards can be constructed in multiple configurations and can contain multiple elements, such as headers, footers, images, icons, links, actions, text and more.

Cards can be placed in a 2, 3, or 4-column layout. Cards in a grid layout should be equal in height and of the same type.

Consider using other formats such as tables, if more than 12 cards are required.

```jsx live
<Card>
	<CardInner>
		<Stack gap={1}>
			<Heading as="h3" type="h4">
				Card title
			</Heading>
			<Text>
				Lorem ipsum dolor sit amet, consectetur adipiscing elit. Cras non
				finibus leo.
			</Text>
		</Stack>
	</CardInner>
</Card>
```

<DoHeading />

- use link cards to provide a contextual link to another page
- limit links to one per card
- avoid images for the most part, unless they add relevant information
- use a pictogram (64 pixels) if it provides meaningful context
- use appropriate alt text for images
- wrap related cards in a list to make navigation easier.

<DontHeading />

- include more than 1 topic per card
- use decorative images
- use formatted/rich text (lists, links etc.) in the card content area
- include more than 3 lines of body copy
- repeat images in card groups
- use different card types in the same grid
- add shadows to non-clickable cards
- add multiple links in the content.

## Images

For a full-width image in a card, add the `img` tag outside the `CardInner` container. For images with inner padding, add the `img` tag inside the `CardInner` container.

```jsx live
<Columns cols={2}>
	<Card>
		<PlaceholderImage />
		<CardInner>
			<Stack gap={1}>
				<Text as="p">Describing content</Text>
				<CardLink href="#">Relevant link</CardLink>
			</Stack>
		</CardInner>
	</Card>

	<Card>
		<CardInner>
			<Stack gap={1}>
				<PlaceholderImage />
				<Text as="p">Describing content</Text>
				<CardLink href="#">Relevant link</CardLink>
			</Stack>
		</CardInner>
	</Card>
</Columns>
```

## Clickable cards

For cards that contain a single link, the hit area for that link can be made to wrap the entire card. This pattern may be used on cards that act as a preview for an article or blog post, for example. These cards could contain an image, a title and a brief summary of the article.

Fully-clickable cards must be given the `clickable` and `shadow` props to make itself and its `CardLink` appropriately indicate the clickable region. Do not wrap cards in an anchor tag as this can be a difficult experience for screen reader users.

```jsx live
<Card clickable shadow>
	<PlaceholderImage />
	<CardInner>
		<Stack gap={1}>
			<H3>
				<CardLink href="#">Title of article</CardLink>
			</H3>
			<Text as="p">Some text</Text>
		</Stack>
	</CardInner>
</Card>
```

## Card footers

Add supplementary information, links or actions to cards by passing a `<CardFooter>` to the `footer` prop.

Use the `footerOutside={true}` prop to render the footer beneath the visible bottom of the card.

A common use-case of `footerOutside` allows for a clickable card to also have separate actions related to the card itself. Do not use `clickable` cards without setting `footerOutside={true}`.

From v1.27.0, you must use the `footer` prop to pass `CardFooter` content to ensure that the “clickable” region doesn’t overlap anything in the footer.

While `Card` still supports `CardFooter` as a child element (as per v1.26.0 and below), it cannot fully support the clickable footer actions use-case without degrading the UX and accessibility, so we recommend migrating to `footer` for all cases.

```jsx live
() => {
	const [isAdded, setIsAdded] = React.useState(false);
	return (
		<Columns cols={2}>
			<Card footer={<CardFooter>Supplementary info</CardFooter>}>
				<CardInner>
					<Stack gap={1}>
						<H3>Card title</H3>
						<Text as="p">Additional conent relating to the card</Text>
					</Stack>
				</CardInner>
			</Card>

			<Card
				footer={
					<CardFooter>
						<ToggleButton
							iconType="star"
							label="Add to favourites"
							onClick={setIsAdded}
							pressed={isAdded}
							pressedLabel="Remove from favourites"
						/>
					</CardFooter>
				}
				footerOutside
			>
				<CardInner>
					<Stack gap={1}>
						<H3>Card title</H3>
						<Text as="p">Additional conent relating to the card</Text>
					</Stack>
				</CardInner>
			</Card>

			<Card
				clickable
				footer={<CardFooter>Supplementary info</CardFooter>}
				shadow
			>
				<CardInner>
					<Stack gap={1}>
						<H3>
							<CardLink href="#">Card title</CardLink>
						</H3>
						<Text as="p">Additional conent relating to the card</Text>
					</Stack>
				</CardInner>
			</Card>

			<Card
				clickable
				footer={
					<CardFooter>
						<ToggleButton
							iconType="star"
							label="Add to favourites"
							onClick={setIsAdded}
							pressed={isAdded}
							pressedLabel="Remove from favourites"
						/>
					</CardFooter>
				}
				footerOutside
				shadow
			>
				<CardInner>
					<Stack gap={1}>
						<H3>
							<CardLink href="#">Card title</CardLink>
						</H3>
						<Text as="p">Additional conent relating to the card</Text>
					</Stack>
				</CardInner>
			</Card>
		</Columns>
	);
};
```

## Equal card heights

The content within each card will often vary in height, so to prevent an uneven, stagged appearance of multiple cards within a list, ensure that the `Card` itself is the immediate child of a flex or grid component, such as `Columns`.

```jsx live
<Columns as="ul" cols={{ xs: 2 }}>
	<Card
		as="li"
		clickable
		footer={
			<CardFooter>
				<TextLink href="#">An action</TextLink>
			</CardFooter>
		}
		footerOutside
		shadow
	>
		<CardInner>
			<Stack gap={1}>
				<H3>
					<CardLink href="#">Learners permit</CardLink>
				</H3>
				<Text>
					A learner driver licence allows you to drive with a supervisor who has
					a full Australian driving licence. A learner driver licence allows you
					to drive with a supervisor who has a full Australian driving licence.
				</Text>
			</Stack>
		</CardInner>
	</Card>

	<Card as="li" clickable shadow>
		<CardInner>
			<Stack gap={1}>
				<H3>
					<CardLink href="#">Provisional licence</CardLink>
				</H3>
				<Text>
					Information on provisional licence applicatios, tests, exemptions,
					changes and checks.
				</Text>
			</Stack>
		</CardInner>
	</Card>

	<Card
		as="li"
		footer={
			<CardFooter>
				<CardLink href="#">An action</CardLink>
			</CardFooter>
		}
	>
		<CardHeader>
			<H3>
				<CardLink href="#">Drivers licence</CardLink>
			</H3>
		</CardHeader>
		<CardInner>
			<Stack gap={1}>
				<Text>
					Get a driver or rider licence, renew, upgrade or change licence and
					licence details.

			</Stack>
		</CardInner>
	</Card>

	<Card>
		<CardInner>
			<Stack gap={1}>
				<PlaceholderImage />
				<Text as="p">
					When starting a business, we try to make it easy to understand and
					provide everything you need to get started.
				</Text>
				<CardLink href="#">Start a business</CardLink>
			</Stack>
		</CardInner>
	</Card>
</Columns>
```

## Colour

AgDS foreground components respond to the background colour of their parent container. When placed on a dark palette background, the dark palette variant of the component is displayed. When placed on a light palette background, the light palette variant is displayed.

This logic ensures sufficient contrast between foreground and background elements is maintained to meet WCAG 2.1 AA 4.5:1 contrast ratio for text (WCAG 1.4.3) and 3:1 for graphic elements that convey information (WCAG 1.4.11).
