## Clear button and controlled input

If you are **controlling the component using the props `value` and `onChange`**, a synthetic event is passed in the argument for `onChange`. The argument will pass the updated input value as `event.target.value`.

The ‘Clear search’ button attempts to provide the required values passed into the `onChange` prop, as we cannot trigger `onChange`. Making a change using `ref.value` can create bugs and [is not recommended by React.](https://react.dev/learn/referencing-values-with-refs#when-to-use-refs)

```jsx
<SearchBoxInput
    ...
    onChange={(event) => setValue(event.target.value)}
/>
```

If you require fields in the `ChangeEvent` that have not been provided by the ‘Clear search’ button, [please contact the Design System team](/about#connect-with-us).
